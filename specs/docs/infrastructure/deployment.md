# Deployment Architecture

## Overview

WinQuake is deployed as a **standalone native binary** alongside game data files. There is no installer framework, no package manager, no containerization, and no cloud deployment. Distribution was via ZIP archives and RPM packages.

## Deployment Method

### Windows Distribution

As described in `wqreadme.txt`, the deployment is:

1. Install Quake base game (provides `id1/pak0.pak`, `id1/pak1.pak`)
2. Unzip WinQuake distribution into the Quake directory
3. Run `winquake.exe` from the Quake directory

**Required files** (from `wqreadme.txt`):
```
c:\quake\
├── winquake.exe        # Main executable
├── pmpro16.dll         # SciTech MGL 16-bit helper
├── pmpro32.dll         # SciTech MGL 32-bit helper
├── wdir16.dll          # SciTech directory helper (16-bit)
├── wdir32.dll          # SciTech directory helper (32-bit)
├── wdirnop.com         # SciTech no-op COM helper
├── wdirnop.pif         # SciTech PIF file
└── id1/                # Game data directory
    ├── pak0.pak         # Base game assets (shareware or registered)
    └── pak1.pak         # Registered game assets
```

### Linux Distribution (RPM)

RPM packages generated by `quake.spec.sh`:

```
/usr/local/games/quake/
├── squake              # SVGALib console binary (setuid root)
├── glquake             # OpenGL binary (setuid root)
├── glquake.glx         # GLX OpenGL binary (setuid root)
├── glquake.3dfxgl      # 3Dfx OpenGL binary
├── quake.x11           # X11 binary
├── README              # Documentation
└── id1/                # Game data
    ├── pak0.pak
    └── pak1.pak
```

RPM file permissions (from `quake.spec.sh`):
- `squake`, `glquake`, `glquake.glx`: mode `4755` (setuid root) — required for SVGALib raw hardware access
- `quake.x11`: mode `755` (normal executable)
- Shared libraries (`lib3dfxgl.so`, `libMesaGL.so.2.6`) installed to `/usr/lib/`

### DOS Deployment
- Single executable (`quake.exe`) with `cwsdpmi.exe` (DPMI host for protected mode)

## Runtime Configuration

### Command-Line Arguments

Configuration is primarily via command-line arguments. Key parameters:

| Argument | Purpose |
|----------|---------|
| `-basedir <dir>` | Set base game directory |
| `-game <dir>` | Set game directory (mods) |
| `-dedicated [n]` | Run as dedicated server with n max players |
| `-listen [n]` | Run as listen server with n max players |
| `-width`, `-height` | Set video resolution |
| `-winsize` | Start in windowed mode |
| `-nosound` | Disable sound |
| `-nocdaudio` | Disable CD music |
| `-nolan` | Disable LAN networking |
| `-nojoy` | Disable joystick |
| `-nomouse` | Disable mouse |
| `-condebug` | Log console output to `qconsole.log` |
| `-heapsize <kb>` | Set memory heap size |
| `-zone <bytes>` | Set zone memory size |

### Configuration Files

Quake executes configuration scripts on startup via `exec quake.rc`:

```
id1/
├── quake.rc           # Master startup config (exec default.cfg, exec config.cfg)
├── default.cfg        # Default key bindings and settings
├── config.cfg         # User settings (auto-saved on quit)
└── autoexec.cfg       # User custom commands (run after config.cfg)
```

Configuration files are plain text containing console commands and cvar assignments:
```
bind "w" "+forward"
sensitivity "3"
volume "0.7"
```

### Batch Launchers (Windows)

`wq.bat` provides preset configurations:

| Command | Configuration |
|---------|--------------|
| `wq fast` | Maximum performance (DirectDraw + DirectSound + DirectInput) |
| `wq safe` | Conservative mode (safe video, wave sound) |
| `wq verysafe` | Minimal mode (no sound, safe video) |
| `wq fastvid` | Fast video, wave sound (lower latency) |
| `wq fastsnd` | Safe video, DirectSound |
| `wq max` | Maximum everything including DirectInput |

## Dedicated Server

The engine can run as a headless dedicated server:
- Windows: `winquake -dedicated` or `sys_wind.c` standalone binary
- Console management via `conproc.c` (separate thread for stdin/stdout on Windows)
- No video initialization, no rendering, no sound
- Server state identified by `cls.state == ca_dedicated`

## System Requirements (Original, from `wqreadme.txt`)

- **OS**: Windows 95 or Windows NT 4.0+
- **CPU**: Pentium-class x86 (i386 assembly required)
- **Memory**: Minimum 5.5MB (`MINIMUM_MEMORY = 0x550000`), Windows minimum 8.5MB, maximum 16MB
- **Disk**: Quake base game installation (~80MB with full data)
- **Optional**: DirectX (for DirectDraw/DirectSound), VESA VBE 2.0 or SciTech Display Doctor
